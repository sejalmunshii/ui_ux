The `` Tag in HTML

Definition

The <img> tag is used to embed images in an HTML document.

Example

<img src="https://example.com/image.jpg" alt="Description of the image" width="300" height="200">

Attributes of the `` Tag

  • src: Path to the image file.
  • alt: Alternative text for accessibility.
  • width: Sets the width of the image (optional).
  • height: Sets the height of the image (optional).
  • title: Tooltip text when hovering over the image (optional).
  • loading: Specifies whether to load the image immediately or defer loading (optional).

Practical Use

The <img> tag is widely used to display images on web pages. It is crucial for:

  • Showing photographs, illustrations, or icons.
  • Providing context for visually impaired users through the alt attribute.
  • Controlling image dimensions for responsive design.

Assignment Questions and Answers

  1. What is the purpose of the `` tag?

    The <img> tag is used to embed images in an HTML document.

  2. What is the significance of the `src` attribute?

    The src attribute specifies the URL or path to the image file.

  3. Why is the `alt` attribute important?

    The alt attribute provides alternative text for the image, which helps with accessibility and displays if the image fails to load.

  4. Can you use the `` tag without the `alt` attribute?

    While it is technically possible, it is highly discouraged as it negatively impacts accessibility.

  5. What do the `width` and `height` attributes do?

    The width and height attributes define the dimensions of the image displayed on the webpage.

  6. What happens if you don't specify `width` and `height`?

    If not specified, the image will display at its original size, which may affect layout.

  7. What is the `title` attribute used for?

    The title attribute provides additional information as a tooltip when hovering over the image.

  8. What is the `loading` attribute?

    The loading attribute indicates whether the image should load immediately or defer loading until needed (lazy loading).

  9. Is the `` tag a block or inline element?

    The <img> tag is an inline element by default.

  10. Can you use CSS to style images?

    Yes, CSS can be used to apply styles to images, such as borders, margins, and effects.

        Relative vs Absolute File Paths
        To make use of HTML file paths, the two options are either relative or absolute file paths. With relative file paths, the files you intend to reference are located in the same folder of your website folder structure. On the other hand, with absolute file paths, you refer to external resources located elsewhere on the web.

        Using absolute file paths, we can access a file content or resource with their full URL path, which means you can also reference content directly from the web. For example, you can refer to an image from a URL like:

        HTML
        img src="https://www.hubspot.com/images/hub.jpg" alt="hub"
        The src is also known as the source attribute, which tells us the file path.

        Alternatively, with relative file paths, the file path references a file relative to the current working location or file in the folder structure. For example, say you  have a hub.jpg file in the images folder, one level up in the folder hierarchy. You can easily access it like this: . The first period represents the current directory, and the next period moves one level up the directory structure.

        For relative file paths, to link to a particular file in the same folder you would use the file name alone, such as hub.jpg. To refer to a file located in a nested images folder, all you need to do is write the name of the folder in front of the path and a forward slash: images/hug.jpg.

        It’s best practice to use relative file paths, as they are local to the folder hierarchy of your project and always remain the same. In contrast, absolute file paths may be located anywhere on the Internet and can change at any time, potentially leading to broken links.

        With relative file paths, care must be taken regarding how you reference these files, the spellings of the file names, their location, file extension, and so on. This is so that you can ensure that the resource loads properly. For absolute file paths, the URL must be valid and contain the needed resource for the web page to load appropriately.

        Also, with relative file paths, images (for example) have to be loaded over the Internet. In some cases, it might take large images some time to load, which can lead to slower websites. Therefore, if you no longer have to depend on an external URL, it’s advisable to use relative file paths over absolute file paths wherever possible.
